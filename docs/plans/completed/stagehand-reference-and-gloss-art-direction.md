# Stagehand Reference And Gloss Art Direction

## Goal

Capture the visual system of `stagehand.dev` as a concrete reference, then translate the useful ideas into a Gloss-specific art direction that fits the product brief.

## Scope

- write one document that reports the Stagehand aesthetic in concrete design terms
- write one document that defines a usable visual direction for Gloss
- connect those docs to the existing frontend/harness documentation so they are discoverable

## Constraints

- the Stagehand report should be descriptive, not imitative
- the Gloss direction must stay aligned with:
  - adult tone
  - calm seriousness
  - literary or academic character
  - depth-first vocabulary, not browser automation
- borrow the structural strengths of the reference without copying its exact motifs

## Validation

- read against `docs/PRODUCT.md`
- read against `docs/FRONTEND.md`
- keep the final docs concrete enough to implement without additional interpretation
